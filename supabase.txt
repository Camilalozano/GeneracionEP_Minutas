# Configuración de Supabase

No guardes credenciales reales en este archivo ni en GitHub.

Configura estos valores en `.streamlit/secrets.toml` para desarrollo local o en
Streamlit Cloud > App settings > Secrets para producción:

```toml
SUPABASE_URL = "https://TU-PROYECTO.supabase.co"
SUPABASE_PUBLISHABLE_KEY = "TU_SUPABASE_PUBLISHABLE_KEY"
SUPABASE_SERVICE_ROLE_KEY = "TU_SUPABASE_SERVICE_ROLE_KEY"
SUPABASE_BUCKET_DOCUMENTOS = "documentos-generados"
ADMIN_PASSWORD = "CAMBIA_ESTA_CLAVE"
```

Notas de seguridad:
- `SUPABASE_SERVICE_ROLE_KEY` permite subir archivos al bucket privado y escribir metadatos sin depender de políticas públicas. Mantenla solo en secretos de Streamlit o variables de entorno.
- `SUPABASE_BUCKET_DOCUMENTOS` puede omitirse si el bucket se llama `documentos-generados`.
- No hagas público el bucket si los documentos tienen información contractual o institucional.

Tablas usadas por la aplicación:
- EstudiosPrevios
- BitacoraAuditoria
- DocumentosGenerados

Bucket usado por la aplicación:
- documentos-generados

## Tabla DocumentosGenerados compatible

La aplicación es compatible con la tabla creada con `id_caso` como columna `identity` y llave primaria. Por eso, al insertar metadatos, la app omite `id_caso` para que Supabase lo genere automáticamente y guarda la trazabilidad del caso dentro de `storage_path`.

Esquema usado:

```sql
create table public."DocumentosGenerados" (
  id_caso bigint generated by default as identity not null,
  storage_bucket text not null,
  storage_path text null,
  nombre_archivo text null,
  mime_type text null,
  size_bytes text null,
  sha256 text null,
  actor text null,
  fecha_hora_utc timestamp without time zone null,
  constraint DocumentosGenerados_pkey primary key (id_caso)
);
```
